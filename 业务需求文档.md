## 2. 项目背景

### 2.1 业务痛点

在传统财税咨询场景中,企业财务人员面临以下痛点:

| 痛点类型 | 具体表现 |
|----------|----------|
| **信息分散** | 财务数据在Excel/ERP系统中,税收政策在各类文件中,查询效率低 |
| **专业门槛高** | 税收优惠政策文件专业性强,非专业人员难以快速理解 |
| **时效性差** | 人工咨询响应慢,无法即时获取财务分析结果 |
| **分析能力弱** | 缺乏自动化的财务数据对比分析和可视化工具 |
| **知识不统一** | 不同渠道获取的税收政策解读可能不一致 |

### 2.2 项目愿景

构建一个**智能化、一站式**的财税对话助手,实现:

- 自然语言交互,降低使用门槛
- 多数据源智能融合,提供精准答案
- 自动化数据分析,提升决策效率
- 政策智能解读,降低合规风险

### 2.3 项目价值

| 价值维度 | 预期收益 |
|----------|----------|
| **效率提升** | 财务数据查询时间从5-10分钟降低到10秒内 |
| **成本降低** | 减少80%的简单咨询人力投入 |
| **体验优化** | 7×24小时智能服务,无需等待 |
| **风险控制** | 统一的政策解读口径,减少误解风险 |

## 3. 业务目标

### 3.1 核心目标

1. **构建智能对话能力**
   - 实现自然语言问答交互
   - 支持财务数据、税收政策、通用问题三类场景

2. **整合多维数据源**
   - 接入本地财务数据库
   - 接入税收优惠政策库
   - 对接扣子智能体知识库

3. **提供数据分析能力**
   - 支持多时间段财务数据对比
   - 自动生成数据可视化图表
   - 提供增长率、变化趋势分析

4. **实现高效用户体验**
   - 快速响应(3秒内开始输出)
   - 精准路由(正确识别用户意图)
   - 便捷导出(支持PDF保存)

### 3.2 量化指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 意图识别准确率 | ≥90% | 正确路由到对应数据源 |
| 首次响应时间 | ≤3秒 | 开始输出内容的时间 |
| 财务查询成功率 | ≥95% | 成功返回数据的查询比例 |
| 用户满意度 | ≥85分 | 百分制满意度评分 |

### 4.2 用户使用场景

#### 场景1: 财务数据查询

**背景**: 财务人员需要快速获取某时间段的财务指标数据

**流程**:

1. 用户输入: "2023年第四季度利润是多少?"
2. 系统识别为财务数据查询
3. 从本地财务数据库检索数据
4. 返回格式化的结果

**期望结果**: 10秒内获取准确数据

---

#### 场景2: 多时间段对比分析

**背景**: 需要对比不同时期的财务表现

**流程**:

1. 用户输入: "对比2023年和2024年各季度的利润变化"
2. 系统识别为对比分析请求
3. 查询多个时间段数据
4. 计算增长率、变化额
5. 生成可视化图表
6. AI总结分析

**期望结果**: 获取详细对比数据、图表和专业分析

---

#### 场景3: 税收优惠政策咨询

**背景**: 企业想了解适用的税收优惠

**流程**:

1. 用户输入: "小微企业有哪些所得税优惠?"
2. 系统识别为税收优惠查询
3. 从政策库检索相关政策
4. AI进行归纳总结
5. 返回结构化政策信息

**期望结果**: 获取完整、准确的政策列表和解读

---

#### 场景4: 税务办理咨询

**背景**: 用户想了解如何办理某项税务事项

**流程**:

1. 用户输入: "如何申报研发费用加计扣除?"
2. 系统识别为知识库问题(非本地数据)
3. 路由到扣子智能体
4. 返回操作指南和注意事项

**期望结果**: 获取清晰的办理流程指引

---

#### 场景5: 导出对话记录

**背景**: 用户需要保存咨询记录作为工作底稿

**流程**:

1. 用户完成多轮对话
2. 点击"导出PDF"按钮
3. 选择保存位置
4. 系统生成包含完整对话和图表的PDF

**期望结果**: 获取格式规范、完整保留图表的PDF文档

## 7. 业务规则

### 7.1 意图识别规则

| BR3.3 | 图表触发 | 查询结果≥2条时自动生成图表 |
